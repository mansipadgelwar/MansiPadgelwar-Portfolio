search & search all in cobol

Search and Search all in cobol is used to search an element in table. Basic difference between Search and Search all is that Search is linear or sequential search and search all is binary search.

  • SEARCH in cobol

    • The search verb can only be applied to a table which has the OCCURS clause and INDEXED phrase. Before the use of SEARCH verb, index should be initialized with some value using SET. The value of index should not exceed size of table otherwise if the AT END clause is specified it will be executed. The SEARCH verb start searching with initial value of index. If none of the mentioned WHEN condition is satisfied the INDEX value is increamented by 1 The process continues until INDEX value exceeds size of the table.

    The WHEN clause indicates what action is to be taken when the condition specified is actually met. -- The AT END clause specifies what should be done if the table has been completely searched and no match is found. -- To use a SEARCH statement, two additional entries are required : the INDEXED BY clause along with OCCURS, and the SET statement in the PROCEDURE DIVISION -- with 2 WHEN clauses the computer begins by performing the first comparison. only if the condition in the first WHEN is not met it test second WHEN
